# Resume-to-DEET Instant Registration System

An AI-powered full-stack web application that reads a job seeker's resume, extracts structured information using NLP, and automatically populates a DEET (Department of Employment, Education, and Training) registration form.

## 🚀 Quick Start

### 1. Prerequisites

- Python 3.9 or higher
- pip
- Tesseract-OCR (for scanned PDFs and images)

**Install Tesseract:**
```bash
# Ubuntu / Debian
sudo apt-get install tesseract-ocr

# macOS
brew install tesseract

# Windows
# Download installer from: https://github.com/UB-Mannheim/tesseract/wiki
```

### 2. Clone / Extract the project
```bash
cd resume_deet
```

### 3. Create a virtual environment
```bash
python -m venv venv

# Linux / macOS
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 4. Install Python dependencies
```bash
pip install -r requirements.txt
```

### 5. Download the spaCy language model
```bash
python -m spacy download en_core_web_sm
```

### 6. Run the application
```bash
cd backend
python app.py
```

The server starts on **http://localhost:5000**

## 🔐 Security Features

1. **Password Policy** — Min 8 chars, 1 uppercase, 1 digit, 1 special character
2. **Password Hashing** — pbkdf2:sha256 via Werkzeug
3. **JWT Authentication** — 24-hour expiry on all protected routes
4. **File Validation** — Extension whitelist + 16MB size cap
5. **Secure Filenames** — werkzeug `secure_filename` + UUID prefix
6. **SQL Injection Protection** — Parameterized queries throughout
7. **CORS** — Configured via Flask-CORS with credentials support
8. **XSS Protection** — HTML escaping in all frontend templates
9. **Error Handling** — No internal stack traces exposed to client

## 🧠 NLP Extraction Pipeline

```
Resume File
    ↓
File Type Detection
    ↓
┌─────────────────────────────────┐
│  pdfplumber (text PDF)          │
│  python-docx (DOCX)             │
│  Tesseract OCR (scanned/image)  │
└─────────────────────────────────┘
    ↓
Text Preprocessing
 - Unicode normalization
 - Whitespace cleanup
 - PDF artifact removal
    ↓
Section Segmentation
 - Header keyword detection
 - Sections: objective, education, skills,
   experience, projects, certifications, etc.
    ↓
Hybrid Extraction
 ┌─────────────────────────────────┐
 │ Regex        → Email, Phone     │
 │ spaCy NER    → Name (PERSON)    │
 │ Rule-based   → Education, Exp   │
 │ Keyword dict → Skills (300+ sk) │
 │ Top-of-doc   → Name fallback    │
 └─────────────────────────────────┘
    ↓
Structured JSON Output
    ↓
Confidence Scoring (per field)
Quality Score (0–100)
```

## 📝 Notes for Hackathon / Final Year Project

- The extraction engine uses a **hybrid approach** — not solely AI or solely regex, but the best of both
- **Name extraction** is specially tuned: first checks spaCy PERSON entities in the top 300 chars, then falls back to the first non-contact line at the top of the resume — matching the DEET template format exactly
- Skills are matched against a **300+ term dictionary** across 5 categories
- All database queries use **parameterized statements** — safe against SQL injection
- The frontend is pure HTML/CSS/JS with **no framework dependency** — easy to run anywhere
